Допущения языка

Допущения языка GDScript весьма банальны, но тем не менее могут вам пригодиться.

Первое - Вы можете не присваивать переменным значения в момент создания, присвоить позже. То есть "var a" без знака равно - абсолютно верная запись. Когда мы говорили о типах, там содержался тип, который не содержит данных - вот, это именно он.

Второе - проверка переменной может быть сокращена за счёт наличия значения переменной, и его логической значимости. То есть "if a == 1:" может быть сокращено до "if a:". Число отличное от нуля считается истинным;

Третье - строку можно сравнить с числом. То есть "if '0' == 0" будет сравнено, и пройдёт проверку на равенство;

Четвёртое - список можно проверить на наличие значений. То есть:

var a = []
if a:
    print('Да') # -> не выведет ничего, так как a пуста

a.append("Что-нибудь")
if a:
    print("Да") # -> Теперь выведет "Да", так как a не пуста, и содержит в себе хотя бы одно значение;

Пятое - тернарные ветвления. Стандартное ветвление записывается в несколько строк. Попробуйте записать ветвление в одну строку, например "x = 5 if a < 0 else 10"

Эти простые, казалось бы, допущения языка, являются одними из тех, которые помогут Вам сильно сократить Ваш код, улучшить его читаемость, и при этом на все эти плюсы использования данных допущений Вы потратите в разы меньше сил при кодировании.
